---
title: "Environment variables"
description: "Configure Firebase credentials and API keys for your deployment"
---

Snippet Bubble Manager uses two configuration mechanisms: a `.env` file for runtime secrets and a `firebase-applet-config.json` file for Firebase initialization. Both must be present before you run the app.

<Warning>
Never commit `.env.local` or `firebase-applet-config.json` to version control if they contain real credentials. Add both files to your `.gitignore` before your first commit.
</Warning>

## Environment variables (.env / .env.local)

Create a `.env.local` file in the project root for local development. The project's `.env.example` documents the variables the app expects:

```bash .env.local
# Required for Gemini AI API calls
GEMINI_API_KEY=your_gemini_api_key_here

# The URL where this app is hosted (used for self-referential links and OAuth callbacks)
APP_URL=http://localhost:3000
```

### Variable reference

| Variable | Required | Description |
| --- | --- | --- |
| `GEMINI_API_KEY` | For AI features | Google Gemini API key. Required for AI snippet generation and inline code completions. Without this key, AI features will silently fail. |
| `APP_URL` | For deployment | The URL where the app is hosted. Used for self-referential links and OAuth callback configuration. Defaults to `http://localhost:3000` in local development. |

To get a Gemini API key, visit [aistudio.google.com](https://aistudio.google.com), sign in with your Google account, and create a new API key.

<Note>
`GEMINI_API_KEY` is accessed via `process.env.GEMINI_API_KEY` in the app code. When deployed to Google AI Studio, this value is injected automatically from your configured secrets. For local development, set it manually in `.env.local`.
</Note>

## firebase-applet-config.json

The app reads `firebase-applet-config.json` from the project root to initialize Firebase and specify the Firestore database. This file is imported directly by `firebase.ts` — it is the primary source of Firebase configuration.

```json firebase-applet-config.json
{
  "apiKey": "your_api_key",
  "authDomain": "your_project.firebaseapp.com",
  "projectId": "your_project_id",
  "storageBucket": "your_project.appspot.com",
  "messagingSenderId": "your_sender_id",
  "appId": "your_app_id",
  "firestoreDatabaseId": "(default)"
}
```

### Field reference

| Field | Required | Description |
| --- | --- | --- |
| `apiKey` | Yes | Firebase Web API key |
| `authDomain` | Yes | Firebase auth domain (format: `project-id.firebaseapp.com`) |
| `projectId` | Yes | Firebase project ID |
| `storageBucket` | Yes | Firebase storage bucket |
| `messagingSenderId` | Yes | Cloud Messaging sender ID |
| `appId` | Yes | Firebase web app ID |
| `firestoreDatabaseId` | Yes | The Firestore database to connect to. Use `"(default)"` for the default Firestore database, or provide a named database ID if you created a non-default database. |

<Steps>
  <Step title="Open the Firebase console">
    Go to [console.firebase.google.com](https://console.firebase.google.com) and select your project.
  </Step>
  <Step title="Navigate to project settings">
    Click the gear icon next to **Project Overview** in the left sidebar, then select **Project settings**.
  </Step>
  <Step title="Locate your web app">
    Scroll to the **Your apps** section. If you haven't registered a web app yet, click **Add app** and choose the web (`</>`) platform.
  </Step>
  <Step title="Copy the config values">
    Under **SDK setup and configuration**, select **Config**. Firebase displays all the values you need. Copy each value into the matching field in `firebase-applet-config.json`.
  </Step>
</Steps>

<Tip>
The project ships with a `.env.example` file listing all required variable names with placeholder values. Commit this file to your repository so teammates know what to configure without exposing real credentials.
</Tip>

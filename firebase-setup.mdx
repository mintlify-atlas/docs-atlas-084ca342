---
title: "Firebase setup"
description: "Configure Firebase Authentication and Firestore for your Snippet Bubble Manager project."
---

Snippet Bubble Manager uses Firebase for two things: Google sign-in via Firebase Authentication, and real-time snippet storage via Firestore. Follow these steps to create your Firebase project and connect it to the app.

<Steps>
  <Step title="Create a Firebase project">
    1. Go to [console.firebase.google.com](https://console.firebase.google.com).
    2. Click **Add project**.
    3. Enter a project name (for example, `snippet-bubble-manager`).
    4. Disable Google Analytics if you don't need it, then click **Create project**.
  </Step>

  <Step title="Enable Google Authentication">
    1. In the Firebase console sidebar, go to **Build > Authentication**.
    2. Click **Get started** if you haven't enabled Auth yet.
    3. Select the **Sign-in method** tab.
    4. Click **Google** under Native providers.
    5. Toggle **Enable** on, enter a public-facing project name and support email, then click **Save**.
  </Step>

  <Step title="Create a Firestore database">
    1. In the sidebar, go to **Build > Firestore Database**.
    2. Click **Create database**.
    3. Choose **Start in production mode** — the app ships with its own `firestore.rules` file.
    4. Select a Cloud Firestore location closest to your users, then click **Enable**.

    <Note>
      The default Firestore database ID is `(default)`. You will use this value in the `firestoreDatabaseId` field of your config file in a later step.
    </Note>
  </Step>

  <Step title="Register a web app and copy the config">
    1. In the Firebase console, click the **gear icon > Project settings**.
    2. Scroll to **Your apps** and click the **Web** icon (`</>`). 
    3. Enter an app nickname (for example, `snippet-bubble-manager-web`) and click **Register app**.
    4. Firebase displays a config object. Copy the values — you'll need them in the next two steps.
  </Step>

  <Step title="Create firebase-applet-config.json">
    Create a file named `firebase-applet-config.json` in the root of the project. Paste in the values from your Firebase config and add the `firestoreDatabaseId` field:

    ```json
    {
      "apiKey": "your_api_key",
      "authDomain": "your_project.firebaseapp.com",
      "projectId": "your_project_id",
      "storageBucket": "your_project.appspot.com",
      "messagingSenderId": "your_sender_id",
      "appId": "your_app_id",
      "firestoreDatabaseId": "(default)"
    }
    ```

    <Note>
      Set `firestoreDatabaseId` to `"(default)"` unless you created a named Firestore database. This value is passed directly to `getFirestore()` in `firebase.ts`.
    </Note>

    <Warning>
      If your `firebase-applet-config.json` contains sensitive API keys, add it to `.gitignore` before committing. The `.env.local` file described below is the safer place to store credentials for version-controlled projects.
    </Warning>
  </Step>

  <Step title="Create .env.local">
    Create `.env.local` in the project root for your runtime secrets:

    ```bash
    GEMINI_API_KEY=your_gemini_api_key_here
    APP_URL=http://localhost:3000
    ```

    Firebase credentials go in `firebase-applet-config.json` (the previous step), not in `.env.local`. The `.env.local` file is for API keys and deployment-specific variables. `.env.local` is excluded from git by default in Vite projects.
  </Step>

  <Step title="Deploy Firestore security rules">
    The project includes a `firestore.rules` file with ownership-based access control. Deploy it to your Firebase project using the Firebase CLI:

    ```bash
    npm install -g firebase-tools
    firebase login
    firebase deploy --only firestore:rules
    ```

    This ensures only the authenticated owner of each snippet can read, write, or delete it.
  </Step>
</Steps>

Once these steps are complete, return to the [Quickstart](/quickstart) to install dependencies and run the app.
